# Godot 客户端接入步骤清单

目标：使用 Godot 做新的游戏客户端，保留当前 Python/FastAPI 后端作为权威规则引擎。Godot 先负责地图、美术、动画和玩家交互，不重复实现世界规则。

## 一、架构原则

- Python 后端继续负责：
  - 世界状态
  - tick 推进
  - LLM 首领
  - 规则校验
  - NPC 执行
  - 上帝能力
- Godot 客户端负责：
  - 地图渲染
  - 像素美术
  - 地块点击交互
  - 事件日志展示
  - 神谕聊天
  - tick 按钮和上帝操作入口
- 第一阶段不要在 Godot 里重写规则引擎，避免两套逻辑不一致。

## 二、创建项目

- 使用 Godot 4.x stable。
- 新建项目：`SimAgentGodotClient`。
- 建议目录：
  - `scenes/`
  - `scripts/`
  - `assets/tiles/`
  - `assets/units/`
  - `assets/ui/`
  - `assets/portraits/`
- 主场景：`scenes/Main.tscn`。
- 主脚本：`scripts/Main.gd`。
- API 脚本：`scripts/ApiClient.gd`。

## 三、最小客户端

- 创建 `Main.tscn`。
- 添加：
  - `Node2D` 作为根节点
  - `Camera2D`
  - `MapRoot`
  - `CanvasLayer` 用于 UI
  - `HTTPRequest` 用于访问后端
- `ApiClient.gd` 先实现：
  - `GET /api/state`
  - `POST /api/tick`
  - `POST /api/god/chat`
  - `POST /api/god/give`
  - `POST /api/god/weather`
  - `POST /api/god/answer`
- 第一目标：Godot 能显示当前 tick、暂停状态、三族资源。

## 四、地图渲染

- 先用 `Node2D + Sprite2D` 渲染每个地块，后续再优化成 `TileMapLayer`。
- 地图 tile 建议 `32x32`。
- 第一批地形图：
  - `plain`
  - `forest`
  - `hill`
  - `water`
  - `mountain`
- 从 `/api/state` 读取每个 tile：
  - `x`
  - `y`
  - `terrain`
  - `owner`
  - `home_of`
  - `weather`
  - `weather_duration`
  - `population`
  - `soldiers`
  - `professions`
  - `houses`
  - `capacity`
  - `protected`
- Godot 内部请求仍使用后端 `x/y`。
- UI 显示坐标时采用玩家视角 `(行, 列)`，也就是显示 `(y, x)`。

## 五、像素美术规格

- 地形 tile：`32x32`。
- 地图单位：`16x16` 或 `24x24`。
- 首领头像：`64x64`。
- UI 图标：`16x16` 或 `24x24`。
- 每个种族固定主色：
  - human：蓝 / 金
  - elf：绿 / 银
  - orc：红 / 暗铁
- 所有像素图使用整数倍缩放。
- 图片导入时关闭模糊过滤，使用 nearest 风格，避免像素发糊。

## 六、地块视觉层

- 底层：地形 tile。
- 第二层：归属边框或半透明领土色。
- 第三层：出生地图标。
- 第四层：天气覆盖层。
- 第五层：保护护盾、人口/士兵/职业图标。
- 天气表现：
  - `rain`：蓝色雨点或轻微蓝色覆盖层
  - `drought`：黄褐色干裂覆盖层
  - `storm`：暗色覆盖层和闪电动画
- 归属表现：
  - human：蓝色边框或小旗
  - elf：绿色边框或小旗
  - orc：红色边框或小旗

## 七、地块详情面板

- 点击地块后显示：
  - 坐标 `(行, 列)`
  - 内部坐标 `x/y`
  - 地形
  - 归属
  - 是否出生地
  - 天气和剩余持续时间
  - 人口
  - 士兵
  - 职业分布
  - 房屋
  - 容量
  - 是否被保护
- 面板只展示后端状态，不在本地推断规则。

## 八、种族面板

- 显示每个种族：
  - 名称
  - 是否淘汰
  - 资源
  - 总人口
  - 总士兵
  - 总房屋
  - 外交关系
  - 最近战略计划
  - 最近规则错误
- 如果某个首领的计划先通过校验，应立刻展示它的计划摘要。

## 九、事件日志

- 使用右侧时间线。
- 显示最近 80 条事件。
- 事件类型建议分色：
  - `tick`：灰色
  - `resource`：金色
  - `population`：绿色
  - `territory`：蓝色
  - `battle`：红色
  - `diplomacy`：紫色
  - `god`：白色或金色
  - `rule_reject`：橙色
  - `elimination`：深红色

## 十、Tick 交互

- 按钮：
  - 推进 1 tick
  - 推进 5 tick
  - 推进 20 tick
- tick 请求期间禁用按钮并显示“世界推进中”。
- 如果后端返回 `409`，提示“世界正在推进，请稍后”。
- 战略 tick 时显示首领思考状态。
- 首领计划通过后先更新该种族卡片，不等所有首领结束。

## 十一、神谕聊天

- 面板包含：
  - 种族选择
  - 输入框
  - 发送按钮
  - 聊天记录
- 调用 `POST /api/god/chat`。
- 发送时禁用按钮，等待首领回复。
- 神谕会进入后端记忆，Godot 不保存权威记忆。

## 十二、上帝能力 UI

- 第一批实现当前 Web 已有能力：
  - 赐资源
  - 改天气
  - 批准祈求
  - 拒绝祈求
  - 神谕私聊
- 第二批再考虑：
  - 保护地块
  - 取消保护
  - 灾害
  - 神迹动画

## 十三、动画优先级

- 第一批：
  - tick 更新淡入
  - 资源增长浮字
  - 人口增长 `+1 idle`
  - 天气覆盖动画
- 第二批：
  - 迁民路线箭头
  - 士兵移动箭头
  - 战斗闪光
  - 占领旗帜替换
- 第三批：
  - 小人 idle 动画
  - 工作动画
  - 攻击动画
  - 首领头像表情

## 十四、开发里程碑

### 里程碑 1：只读世界

- Godot 能启动。
- 能请求 `/api/state`。
- 能显示完整地图。
- 能显示 tick、暂停状态和三族资源。
- 能点击地块查看详情。

### 里程碑 2：可推进世界

- Godot 能调用 `/api/tick`。
- tick 期间有 loading 状态。
- tick 后刷新地图、资源、人口、事件。
- 战略回合时能看到首领计划摘要。

### 里程碑 3：神谕和上帝操作

- 能发送神谕。
- 能显示首领回复。
- 能赐资源。
- 能改天气。
- 能批准/拒绝祈求。

### 里程碑 4：像素美术替换

- 地形 tile 替换完成。
- 种族旗帜/边框完成。
- 天气覆盖完成。
- 人口、士兵、职业图标完成。

### 里程碑 5：游戏感增强

- 迁民、战斗、占领、天气、资源增长有动画反馈。
- 事件日志按类型分色。
- 首领计划展示更清楚。
- 旧 Web 前端可以进入维护模式。

## 十五、暂时不要做

- 不要一开始把 Python 规则引擎迁到 Godot。
- 不要一开始做复杂战斗动画。
- 不要一开始做大量角色动画。
- 不要一开始做复杂 UI 皮肤。
- 不要让 Godot 本地保存权威世界状态。
- 不要让 Godot 自己推断合法性，合法性仍以后端规则引擎为准。

## 十六、第一步建议

先完成最小只读客户端：

1. Godot 项目启动。
2. 请求 `/api/state`。
3. 渲染 `32x20` 地图。
4. 点击地块显示详情。
5. 显示最近事件。

这一步完成后，再接 tick 和神谕。这样风险最低，也最容易看到美术效果。
